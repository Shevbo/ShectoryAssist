# ADR 0001: исходящее аудио в Max и парсинг Gazeta.ru

## Контекст

Требуется сценарий «голос в → голос из» (ТЗ, U2). Max Bot API отправляет медиа через двухшаговый upload (`POST /uploads` → загрузка → `POST /messages` с вложением). Источник контента в MVP — HTML главной Gazeta.ru.

## Решение

1. **Исходящее аудио:** при наличии PCM/MP3 от Gemini TTS выполняется `api.uploadAudio`, затем отправка сообщения в чат с вложением `audio` и текстом (краткое дублирование для клиентов без автопроигрыша). При любой ошибке TTS или upload — ответ только текстом.
2. **Парсинг:** выделенный слой селекторов в `packages/skills/gazeta` + тест на фикстуре. Глобальный лимит запросов к сайту и TTL кэша HTML снижают нагрузку (см. ТЗ §7).
3. **TTS-модель:** задаётся переменной `GEMINI_TTS_MODEL`; при смене API Google поведение проверяется на стенде, при необходимости обновляется runbook.

## Статус

Принято для MVP 0.1.

## Последствия

- Необходимы ретраи при `attachment.not.ready` (реализуются в будущих итерациях при необходимости).
- Селекторы HTML хрупки — мониторинг `parseEmpty` в логах.
