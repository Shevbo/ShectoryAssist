# ADR 0003: транспорт Telegram вместо Max

## Контекст

Интеграция с мессенджером **Max** временно приостановлена; нужен рабочий канал для MVP сценария «голос → навык → голос».

## Решение

1. Вход/выход реализованы через **Telegram Bot API** и библиотеку **grammy** в `apps/bot`.
2. Пакет `packages/adapters/max` **удалён** из репозитория (не публикуется в сборке); возврат к Max предполагает восстановление адаптера по git-истории или новый PR.
3. Webhook: `POST /telegram/webhook`, проверка `X-Telegram-Bot-Api-Secret-Token` при наличии `TELEGRAM_WEBHOOK_SECRET`.
4. Идемпотентность оркестратора: ключ `chatId:message_id` (Telegram), `traceId`: `tg-<update_id>`.

## Последствия

- ТЗ и продуктовая документация по-прежнему описывают Max как целевую платформу — при возобновлении работы с Max нужно синхронизировать документы или оформить отдельное ADR «двухканальность».
